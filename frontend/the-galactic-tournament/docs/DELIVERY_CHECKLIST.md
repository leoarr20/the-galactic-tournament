# Delivery Checklist

Use this checklist before sending the project or presenting it in the interview.

## Backend

- [ ] API is running locally.
- [ ] Database exists and migrations were applied.
- [ ] Swagger opens correctly.
- [ ] `GET /api/species` works.
- [ ] `POST /api/species` works.
- [ ] `POST /api/battles` works.
- [ ] `POST /api/battles/random` works.
- [ ] `GET /api/ranking` works.
- [ ] CORS allows `http://localhost:4200`.

## Frontend

- [ ] Angular runs with `ng serve`.
- [ ] Home page explains rules.
- [ ] Species page registers and lists species.
- [ ] Battles page starts selected battles.
- [ ] Random battle works with at least two species.
- [ ] Battle history is visible.
- [ ] Ranking updates after battles.
- [ ] UI works on smaller screens.

## Repository

- [ ] `node_modules` is not committed.
- [ ] `dist` is not committed.
- [ ] README includes installation and execution steps.
- [ ] Technical decisions are documented.
- [ ] No real production passwords are committed.
